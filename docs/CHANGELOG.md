# Changelog

이 문서는 실제 변경 이력을 누적한다.

## Unreleased

- `Studio YONA HomePage` 워크스페이스 초기 세팅을 추가했다.
- 기본 문서 세트(`README`, `DOCS_RULES`, `CHANGELOG`, `ARCHITECTURE`, `worklog template`, `initial worklog`)를 생성했다.
- `Studio YONA` 회사 홈페이지의 MVP 기획안 문서를 추가했다.
- MVP 기획안 리뷰를 반영해 앱별 법률 URL 구조, `www` canonical 도메인 규칙, Gmail 기반 문의 처리 고지 원칙을 문서에 반영했다.
- 정적 사이트 MVP 1차 구현으로 공통 페이지, `EDSN Frame` 앱 상세, 앱별 정책 페이지, 공통 자산을 추가했다.
- 리뷰 반영으로 neutral 법률 URL 전략과 이메일 첨부 처리 고지를 문서와 정책 페이지에 반영했다.
- 일반 페이지를 한국어 우선 루트 경로와 `?lang=en` 토글 구조로 재정렬하고, 기존 `/ko/...`, `/en/...` 일반 페이지는 호환 래퍼로 전환했다.
- 앱 상세/법률 페이지의 중복 인라인 CSS를 `assets/site.css`로 통합하고, 앱 관련 일반 링크를 최신 루트 경로 구조에 맞게 정리했다.
- 배포 문서를 현재 구조에 맞게 갱신하고, GitHub Pages 설정/DNS/배포 후 확인용 quick checklist를 추가했다.
- `summerjelly` GitHub 계정과 `호스팅케이알` 기준의 DNS 클릭 가이드를 배포 문서에 추가했다.
- 일반 페이지의 언어별 헤더/섹션 중복 노출을 막기 위해 공통 CSS에 `[hidden]` 강제 숨김 규칙을 추가했다.
- 홈 화면을 더 미니멀하게 정리하고, 일반 페이지 영문 헤더 링크가 `?lang=en` 상태를 유지하도록 수정했다.
- 새 앱 `탐정의 녹음기`의 한/영 상세 페이지와 앱별 법률 URL을 추가하고, 앱 목록에서 새 페이지로 진입할 수 있게 연결했다.
- 홈을 레퍼런스 무드에 맞춘 원페이지 랜딩으로 재구성하고, 작은 언어 전환과 햄버거 오버레이 메뉴, 감성적인 카피 배치, 시네마틱 섹션 흐름을 적용했다.
- 로컬 미리보기 아이콘이 Safari 캐시에 막히지 않도록 매 실행마다 cache-busting query를 붙여 새 창을 열게 조정했다.
- MVP 기획안을 현재 구현 상태에 맞춰 원페이지 홈, `탐정의 녹음기` 추가, 로컬 미리보기 흐름까지 반영해 업데이트했다.
- MVP 기획안에 홈 메뉴 항목, 앱 섹션 구성 방식, 회사명 우선 히어로, 이미지/그라디언트 배경, 시네마틱 전환 등 사용자가 확정한 리디자인 세부 방향을 명시적으로 보강했다.
- 기획안 리뷰 반영으로 콘텐츠 원본 페이지 규칙, 영어 일반 페이지 canonical 운영 기준, 디자인 수동 검수 완료 조건을 추가했다.
- 기획안 완료 조건 기준 gap closure 루프로 404 페이지를 현재 루트/영문 토글 구조에 맞게 정리하고, 로컬 미리보기 런처가 실제로 `8123` 서버를 유지하도록 보강했다.
- 수동 QA에서 나온 홈/상세 페이지 refinement backlog를 MVP 기획안에 반영하고, 오버레이 메뉴 크기, `Tiny app studio`, 헤더 겹침, Safari overflow, 줄바꿈 목표, 축약형 헤더 일관성, 추상 비주얼 해석 원칙을 문서화했다.
- 홈의 오버레이 메뉴를 더 절제된 크기로 조정하고, `Tiny app studio`, 고정 줄바꿈, 헤더 겹침 방지, Safari overflow 대응을 반영했다.
- 일반 페이지와 앱 상세 페이지를 홈과 같은 계열의 축약형 헤더 + 햄버거 오버레이 메뉴 구조로 정리해 예전식 상단 카테고리 내비게이션 노출을 제거했다.
- 홈 한국어 핵심 카피와 앱 제목 줄바꿈이 데스크톱/확대 화면에서도 의도대로 유지되도록 폭 제한과 white-space 규칙을 보강했다.
- 홈 한국어 핵심 카피 3종의 줄바꿈 목표를 다시 조정해 기획안과 실제 홈 마크업에 같은 줄 단위로 반영했다.
- 홈 `Contact` 제목 줄바꿈 목표를 다시 조정하고, 회사 소개 첫 비주얼 카드의 `Studio YONA` 타이포를 가운데 정렬로 보정했다.
- 회사 소개 첫 비주얼 카드가 화면에서 더 중앙에 오도록 카드 블록 자체의 폭과 정렬을 보정했다.
- 홈 첫 히어로 카드가 화면 중심에 더 가깝게 보이도록 정렬을 보정하고, 주요 비주얼 카드/섹션 사이 세로 여백을 넓혔다.
- Safari 캐시 영향을 줄이기 위해 공통 `site.css`/`site.js`에 버전 쿼리 스트링을 붙였고, 앱 법률 페이지까지 축약형 헤더 + 햄버거 구조로 통일했다.
- 앱 상세 페이지의 앱 이름은 `app-lockup` 구조로 바꿔 줄바꿈이 흔들리지 않도록 고정했다.
- 앱 상세 첫 화면에서 앱 이름이 잘려 보이지 않도록 제목 최대 폭과 폰트 크기를 다시 조정하고, 작은 화면에서는 `nowrap` 제약을 풀었다.
- 햄버거 메뉴 오버레이의 패널 폭, 패딩, 버튼 높이, 링크 폰트 크기를 한 단계 더 줄여 전체적으로 더 컴팩트하게 보이도록 조정했다.
- 햄버거 메뉴 오버레이의 패널 폭을 다시 한 단계 더 줄여 더 슬림하게 보이도록 조정했다.
- 햄버거 메뉴 오버레이의 내부 패딩과 링크 패딩도 함께 더 줄여 전체적으로 더 압축된 컴팩트 버전으로 조정했다.
- Safari 캐시 대응을 위해 기존 아이콘과 별개로 새 포트를 쓰는 Safari 전용 미리보기 아이콘 경로를 준비했다.
- 햄버거 메뉴를 한 단계 더 줄여 패널 폭, 상단 간격, 링크 높이, 폰트 크기 모두 추가로 축소했다.
- 햄버거 메뉴는 폰트 크기를 유지한 채 내부 패딩과 링크 여백만 더 줄여 더 촘촘한 버전으로 조정했다.
- Safari 전용 로컬 미리보기 아이콘은 매 실행마다 사용 가능한 새 포트에서 프로젝트를 열도록 바꿔, Safari에서 예전 화면이 반복 노출되는 문제를 줄였다.
- 홈, 일반 페이지, 앱 상세/정책 페이지 최하단에 Studio YONA 사업자 정보와 통신판매 번호를 노출하는 푸터 정보 블록을 추가했다.
- 사업자 정보는 링크 푸터와 시각적으로 분리된 작은 하단 정보줄처럼 보이도록 크기와 여백, 구분선을 다시 조정했다.
- 홈 원페이지에서는 사업자 정보 블록을 `Contact` 섹션 내부에서 분리해, 페이지 맨 아래의 별도 하단 구역으로 재배치했다.
- 홈 하단 사업자 정보 구역은 더 작은 패딩과 약한 그림자, 작은 글씨를 적용해 얇은 footer strip처럼 다시 다듬었다.
- 홈 하단 사업자 정보 구역은 공통 카드 스타일을 제거해, 박스 없는 얇은 footer strip처럼 다시 정리했다.
- 홈 햄버거 메뉴의 `회사 소개`는 상세 `/about/`로 연결하고, 홈 About 요약 섹션에도 상세 페이지로 이동하는 버튼을 추가했다.
- 모든 페이지의 하단 별도구역을 홈 기준 footer strip으로 통일하고, 법률 링크는 제거한 채 `© Studio YONA + 사업자 정보`만 남기도록 정리했다.
- 모든 페이지 footer strip의 상단 여백을 늘려, 구분선이 바로 위 카드/박스와 붙어 보이지 않도록 조정했다.
- footer strip은 `padding-top`이 아니라 `margin-top`으로도 간격을 확보해, 구분선 자체가 위 박스에서 더 떨어져 보이도록 다시 조정했다.
- footer strip 하단 패딩을 줄여, 주소 아래쪽에 남는 빈 공간이 덜 보이도록 조정했다.
- GitHub 웹 업로드와 GitHub Pages 설정, 호스팅케이알 DNS 연결을 통해 `www.studioyona.co.kr` 공개 배포를 완료했다.
- 무료 GitHub Pages public 운영 유지 여부와 GitHub Desktop push 오류를 미해결 운영 이슈로 문서화했다.
- README와 배포 문서를 현재 공개 운영 상태 기준으로 갱신하고, 이후 수정/배포 운영 절차를 정리했다.
- GitHub Desktop push 문제를 다시 진단한 결과, 휴지통 복구 이슈보다 GitHub 저장소 공개 접근 상태 이상 가능성이 더 높다는 최신 판단을 문서에 반영했다.
- GitHub Desktop/Git fetch 문제와 public 저장소 404 현상에 대해 GitHub Support 티켓을 제출했고, 후속 응답 대기 상태를 문서에 반영했다.
- GitHub Support 답변으로 저장소명이 `studioyona-hompage`였다는 오타 원인을 확인했고, SSH remote를 교정해 Git fetch가 정상 동작하도록 정리했다.
