## Goal
- 참고 사이트 무드에 맞춰 홈을 원페이지 랜딩 형식으로 재구성한다.
- 상단 메뉴 노출 없이, 작은 언어 전환과 햄버거 오버레이 메뉴만 남긴다.
- 긴 스크롤 안에서 회사 소개, 앱 소개, 고객지원, 문의하기를 감성적인 카피 배치로 보여준다.

## Scope
- 포함 범위:
  - 원페이지 홈 리디자인
  - 오버레이 메뉴와 섹션 앵커 이동
  - 줄바꿈 제어와 시네마틱 전환
- 제외 범위:
  - 상세 페이지 전면 리디자인
  - 앱별 법률 문서 내용 변경

## Plan
1. 홈을 원페이지 섹션 구조로 재설계한다. -> verify: 홈/회사 소개/앱소개/고객지원/문의하기 섹션이 한 페이지에 존재한다.
2. 상단 UI를 햄버거 오버레이 메뉴와 작은 언어 전환으로 바꾼다. -> verify: 초기에는 메뉴가 숨겨지고, 버튼 클릭 시 섹션 이동 메뉴가 열린다.
3. 감성적 카피 배치, 큰 타이포, 반응형 줄바꿈, 배경/전환 효과를 적용한다. -> verify: 데스크탑/태블릿/모바일에서 문장 흐름이 다르게 제어된다.
4. 기존 상세/법률 페이지 연결을 유지한다. -> verify: 각 앱의 상세/법률 링크는 그대로 동작한다.

## Findings
- 현재 홈은 멀티페이지 진입 허브 구조라 참고 사이트 같은 원페이지 몰입감이 약하다.
- 사용자는 법률 링크를 상단이 아니라 하단 또는 앱 섹션 쪽으로 내리고 싶어 한다.
- 홈만 원페이지로 재구성해도 기존 상세/법률 페이지를 그대로 유지할 수 있어 구현 리스크가 낮다.

## Decisions
- 결정: 홈만 강하게 리디자인하고, 나머지 일반 페이지와 앱 상세 페이지는 일단 유지한다.
  이유: 레퍼런스 무드를 빠르게 반영하면서도 현재 정보 구조와 App Store 대응 링크를 보존하기 위해서다.

## Changes
- `index.html`을 원페이지 랜딩 구조로 교체했다.
- 상단은 작은 언어 전환과 햄버거 메뉴만 남기고, 전체 화면 오버레이 메뉴에서 섹션 앵커 이동이 가능하도록 설계했다.
- 히어로, 회사 소개, 앱 소개, 고객지원, 문의하기를 한 페이지 안에 재배치하고 `EDSN Frame`을 먼저 강조했다.
- 반응형 줄바꿈 제어용 클래스와 그라디언트/노이즈/오브 기반 배경, 시네마틱한 reveal 흐름을 홈 전용 스타일로 추가했다.
- 햄버거 메뉴의 열기/닫기, ESC 닫기, 메뉴 클릭 후 닫기 동작을 `assets/site.js`에 추가했다.
- 기존 앱 상세/법률 링크는 그대로 유지하고, 상단 대신 각 앱 섹션과 하단 문의 영역에서 접근하게 정리했다.

## Verification
- `python3 -m http.server 8125`로 로컬 서버를 띄우고 홈 페이지 응답을 확인했다.
- `curl -I http://127.0.0.1:8125/`로 홈이 `200 OK`인지 확인했다.
- `curl -s http://127.0.0.1:8125/ | rg 'id=\"home\"|id=\"about\"|id=\"apps\"|id=\"support\"|id=\"contact\"|data-menu-toggle|data-menu'`로 원페이지 섹션/메뉴 마크업이 노출되는지 확인했다.
- `node --check assets/site.js`로 공통 스크립트 문법 오류가 없는지 확인했다.
